Version Log


Version 1.0.7 - 4/7/2022 (Current)
  • Fixed a bug with the basic TFL elimination rules (Conjunction, Disjunction, Implication, and Biconditional Elimination). These rules now check that the root operand on line m properly matches the rule being applied.
Version 1.0.6 - 3/13/2022
  • Added an "Upload Image" option to the Report Bug/Feedback form so that users can send screenshots when reporting their feedback
Version 1.0.5 - 3/10/2022
  • Added line re-ordering functionality to the Proof form.
    • Added an "Up Arrow" button which swaps the current row with the one above if they're sequential or pushes current row up into the previous row's level.
    • added a "Down Arrow" button which swaps the current row with the one below it if they're sequential or pushes current row down into the next row's level.
  • Added hover text to the left-menu icons when the menu is collapsed.
Version 1.0.4 - 3/10/2022
  • Added Assignments functionality:
    • Instructors can create Assignments for a course that can be viewed by all Students enrolled in that course.
    • Instructors can add Problems to an assignment. A Problem is essentially an incomplete proof with additional attributes ("points" and "target steps").
    • Students can access Assignments for courses in which they are enrolled.
    • Students can attempt to solve Assignment problems and save their solutions as proofs.
Version 1.0.3 - 3/6/2022
  • Added automatic updating of line numbers in citations.
    • As users add and delete lines within a proof, citations in the "rule" field will change to reflect the new, updated line number.
    • Line numbers also update when lines are moved into or out of a subproof
  • Added a "Courses" component to the system.
    • Instructors can create courses, specifying a course title, term, and section.
    • Instructors can enroll students in their courses.
    • Instructors can view proofs created by students in their courses.
    • Students can view courses they are enrolled in.
    • Students can join available courses in which they are not already enrolled.
  • Added a "Report Bug / Feedback" page that allows users to send bug reports and additional feedback to the development team.
Version 1.0.2 - 3/3/2022
  • Proofchecker will now recognize "Fxab" as validate predicate syntax in FOL (in addition to "F(x, a, b)").
  • The escape commands for symbols in proof fields have been updated as follows:
    • Implies: \imp
    • Contradiction: \con
    • Universal Quantifier: \all
    • Existential Quantifier: \exi
Version 1.0.1 - 2/27/2022
  • ProofChecker will now validate the restriction "c must not have appeared earlier in the proof" for rule "∃E".
  • ProofChecker will now validate the restriction "B must not contain c" for rule "∃E".
  • ProofChecker will now validate the restriction "c must be a generic free variable" for rule "∀I".
  • Fixed a bug where the CQ rule was not verifying whether the variables and domains on two lines were the same.
  • Split the TFL Rules and FOL Rules into separate tabs in the right-hand navigation.
  • Added a counter to the Create/Edit Proof pages that displays the number of steps in the proof.
Version 1.0.0 - 2/27/2022
  • Fixed a bug with the basic FOL rules preventing verification of sentences containing multiple predicates.
    • Note: ProofChecker currently does not validate the restriction "c must not have appeared earlier in the proof" for rule "∃E".
    • Note: ProofChecker currently does not validate the restriction "c must be a generic free variable" for rule "∀I".
  • Added a "Students Proofs" page for Instructor accounts so that instructors can view proofs created by students.
  • Renamed the "Proofs" page to "My Proofs" for clarity.
  • Added pagination on the "My Proofs" page to prevent Proof cards from overflowing the boundaries of the page.
  • Added a "Download" button on the Create/Edit Proof pages that allows the user to download a PDF file containing their proof.
  • Added a note under the ∀I image in the Rules tab explaining that "c must be a generic free variable".
